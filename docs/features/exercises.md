# Exercises

## Types
| type | description |
|------|-------------|
| `multiple-choice` | 4 options, one correct |
| `fill-in-blank` | type the missing word/phrase |
| `true-false` | statement is correct or not |
| `rewrite` | rewrite sentence using a given structure |

## Sources
- `static` — pre-written and seeded in DB during import or manually added
- `ai` — generated by Claude API (Phase 2)

## Static exercise seeding (Phase 1 plan)
For MVP: manually write 5–10 exercises per unit for the first 30 units.
Store as SQL seed script in `scripts/seed-exercises.sql`.
Run once after PDF import.

## AI exercise generation (Phase 2)
`claude:generateExercises(chapterId, count)` calls Claude with unit's raw text.
Response parsed as JSON array. Stored in `exercises` table with `source = 'ai'`.
Requires user's Claude API key in settings.

## Scoring
- Each exercise: correct = 1 point
- Chapter score = (correct / total) × 100
- Score ≥ 70 → chapter marked complete, next unit unlocked

## Files
- Types: `src/shared/types.ts` → `Exercise`, `ExerciseType`
- IPC: `src/main/ipc/database.ts` → `db:getExercises`
- AI generation: `src/main/services/ai.ts` → `generateExercises`
- UI: `src/renderer/pages/Exercise.tsx`
